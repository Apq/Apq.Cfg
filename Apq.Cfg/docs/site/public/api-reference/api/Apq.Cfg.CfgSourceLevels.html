Remarks

层级规划原则：数值越大优先级越高，高层级覆盖低层级

| 层级范围 | 用途 | 配置源 | |----------|------|--------| | 0 | 基础配置文件 | Json, Ini, Xml, Yaml, Toml | | 100 | 远程存储 | Redis, Database | | 200 | 配置中心 | Consul, Etcd, Nacos, Apollo, Zookeeper | | 300 | 密钥管理 | Vault | | 400 | 环境相关 | .env, EnvironmentVariables（最高优先级覆盖） |
